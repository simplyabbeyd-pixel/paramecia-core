# 🌿 Paramecia / Triquel Canon Index

> **Purpose:** Provide the authoritative navigation and status layer for Paramecia, Triquel, Whisperwood, Hells Branch, characters, institutions, scenarios, recovery systems, and derived artifacts.

> **Core axiom:** **Nothing flourishes alone.**

> **Core civic rule:** **Observation does not create authority. Care is infrastructure.**

## 1. Source Authority

1. **GitHub versioned canon ledger** — authoritative, reviewable, and rollback-capable.
2. **Notion world studio** — readable synthesis, navigation, and active development.
3. **Project chats and source artifacts** — exact language, corrections, discoveries, and provenance.
4. **Slides, images, JanitorAI exports, PDFs, and other derived artifacts** — expressions of canon, never independent authority.

Every canon entry should record:

- stable identifier
- canon status
- layer and scope
- source or provenance
- aliases and deprecated names
- correction history
- relationship links
- review date or version

## 2. Canon Status Vocabulary

- **Canon Locked** — authoritative and stable until formally revised
- **Canon Supported** — consistent with canon but interpretive or incomplete
- **Provisional** — active development awaiting review
- **Exploration** — generative possibility without authority
- **Contradiction** — conflicts with a higher-authority source or correction
- **Deprecated** — intentionally retired but preserved for provenance
- **Archive** — historically relevant and inactive

Corrections are first-class canon events. Never erase a correction trail during consolidation or export.

## 9. Integration Rule

Before creating a new master document:

1. search the existing ledger
2. extend an existing node where possible
3. identify the affected layer
4. mark canon status
5. list relationship effects
6. preserve source anchors and corrections

The goal is not one enormous lore document.

The goal is one coherent, navigable organism whose parts can disagree, evolve, and still recognize one another.
