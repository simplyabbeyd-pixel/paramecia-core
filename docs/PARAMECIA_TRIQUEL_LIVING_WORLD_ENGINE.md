# Paramecia / Triquel Living World Engine

> **Core axiom:** Nothing flourishes alone.

This document consolidates the current conceptual architecture of Paramecia and Triquel into a single versioned reference for presentation design, worldbuilding, simulation design, and canon governance.

## 1. Executive summary

Paramecia is not merely a fantasy setting. It is a relational civilization engine.

Its design asks a world to simulate:

- ecology
- architecture
- relationships
- memory
- consent
- repair
- civic institutions
- cultural evolution
- travel and exploration
- personhood
- environmental change

The result is a world where places remember, relationships alter institutions, care is infrastructural, and player choices continue producing consequences beyond the immediate scene.

## 2. Core laws

### 2.1 Relation is ontology

Entities do not become meaningful in isolation. Identity, place, memory, and agency emerge through relation.

### 2.2 Care is infrastructure

Care is not flavor text or a private virtue. It appears in rooms, civic roles, procedures, recovery spaces, public expectations, and repair systems.

### 2.3 Consent must remain legible

Consent is supported by visible signals, explicit permissions, revisable agreements, clear exits, and witnessed terms.

### 2.4 Repair changes the world

Repair is not a reset button. Successful repair produces new memory, altered trust, revised institutions, and changed future options.

## 3. Cosmology and world model

### Paramecia

The larger universe of relational possibility, emergence, contradiction, memory, and becoming.

### Triquel

The lived world where those forces become ecology, culture, architecture, institutions, kinship, and play.

### The Five Paths

The Five Paths act as recurring modes of inquiry rather than rigid factions:

1. **Discovery** — curiosity, exploration, mapping, encounter
2. **Wisdom** — memory, pattern, reflection, continuity
3. **Craft** — making, repair, embodiment, material knowledge
4. **Wonder** — imagination, transformation, delight, strangeness
5. **Possibility** — thresholds, synthesis, contradiction, becoming

## 4. The Three Accords

The Accords are civic questions used to evaluate decisions.

### Harmony Accord

**What wants to flourish?**

Examines growth, mutual benefit, vitality, and sustainable participation.

### Midnight Accord

**What must be honored?**

Protects grief, limits, history, silence, debt, endings, and truths that cannot be brightened into disappearance.

### Recognition Accord

**What is actually here?**

Requires accurate observation before interpretation, intervention, or authority.

Together, the Accords create a three-part ethical frame:

1. See clearly.
2. Honor what is present.
3. Support what may flourish.

## 5. Point 0 and the Circle of Kin

Point 0 represents attention before specialization.

It is not ownership, command, or superiority. It is the central coordinate from which relation becomes visible.

The Circle of Kin expresses differentiated attention:

- **Foxkin** — possibility
- **Deerkin** — pattern
- **Ravenkin** — significance
- **Nestkin / Housefolk** — belonging
- **Rabbitfolk** — vigilance
- **Ratfolk** — utility
- **Hedgekin** — regulation

The center is not inside the circle as a ruler. It is the shared coordinate around which the circle becomes legible.

## 6. Living architecture

Architecture in Triquel is active social technology.

A living place may:

- remember how it was treated
- reveal or conceal routes
- respond to stress
- preserve consent boundaries
- support recovery
- reorganize civic behavior
- become wounded
- be repaired through relationship

### Design rule

Every major location should define:

1. ecological condition
2. inhabitants
3. originating need
4. material manifestation
5. social function
6. failure state
7. restoration path
8. playable consequence

### Key examples

- Riverforge
- Whisperwood

- Shelf of Unfinished Things

## 7. Consent and communication protocols

The world uses explicit color states to reduce ambiguity under intensity:

- **Green** — continue
- **Yellow** — slow down, clarify, renegotiate
- **Purple** — curiosity, not automatic consent
- **Red** — stop
- **White** — rest
- **Amber** — overload, recovery, or aftercare

Variants can communicate mixed states, including Purple-Green and Purple-Yellow.

### Protocol principles

- Observation does not create authority.
- Permission is revisable.
- Exits remain visible.
- Silence is not automatically agreement.
- Repair requires recognition of impact.
- Aftercare is part of the interaction, not an optional epilogue.

## 8. Relationship infrastructure engine

Relationships should operate at three interacting layers.

### Surface layer: visible protocol

Players can see and act on:

- consent states
- promises
- boundaries
- requests
- repair offers
- witnessed agreements

### Middle layer: social memory

The system tracks:

- trust
- rupture
- reliability
- affection
- fear
- debt
- recognition
- unresolved contradiction

### Civic layer: institutional consequence

Relationships can alter:

- access to rooms
- faction behavior
- civic roles
- laws and procedures
- architecture
- festivals
- routes
- resource circulation
- available quests

### Recommended hybrid model

Keep consent and recovery protocols visible at the surface. Track trust and memory semi-explicitly beneath them. Allow institutions and locations to reflect accumulated consequences.

## 9. Campaign runtime

The campaign engine should treat player action as a persistent input into the world.

### Runtime loop

1. Observe a living situation.
2. Identify needs, histories, and active relationships.
3. Choose an intervention.
4. Resolve immediate effects.
5. Update social memory.
6. Update ecological or architectural state.
7. Update institutional response.
8. Generate new opportunities, tensions, and routes.

### World-state categories

- NPC memory
- relationship state
- faction state
- civic state
- ecological state
- architectural state
- unresolved promises
- repair obligations
- cultural change

## 11. Canon pipeline

The recommended pipeline is:

**Seed → Attractor → Candidate → Canon → Revision**

### Seed

A phrase, image, note, scene, color, motif, or possibility.

### Attractor

A recurring pattern that begins drawing related material toward itself.

### Candidate

A coherent version proposed for formal inclusion.

### Canon

A version confirmed by source, correction history, consistency, and deliberate acceptance.

### Revision

A controlled change that preserves provenance and records what was replaced.

### Canon governance requirements

Every canon item should track:

- canonical name
- aliases
- status
- source
- first appearance
- last correction
- related entities
- visual lock
- contradiction notes
- revision history

## 13. Current risks

### Missing primary-source consolidation

Project chats, Drive files, image notes, character dossiers, and prior codex material still need to be indexed into one source map.

### Canon drift

Aliases, duplicate exports, visual corrections, and parallel versions can quietly split canon.

### Character under-specification

Some characters are richly visual but still need explicit dramatic arcs.

### Abstract mechanics

The relationship engine must be demonstrated through concrete scenes, consequences, and state changes.

### Decorative architecture

Living architecture must remain operational rather than becoming atmosphere-only worldbuilding.

## 14. Production roadmap

### Stage 1 — Source recovery

- inventory project chats
- inventory Drive and uploaded files
- identify duplicates
- establish source IDs
- preserve corrections

### Stage 2 — Canon registry

- build entity index
- register aliases
- label provisional and confirmed canon
- record visual locks
- record contradictions

### Stage 3 — Runtime specification

- define relationship states
- define world-state updates
- define location memory
- define institutional consequences
- build one end-to-end example

### Stage 4 — Presentation and guidebook

- expand the research deck
- add diagrams
- add character and location visuals
- create speaker notes
- create a shorter executive version

### Stage 5 — Playtest

- run one settlement crisis
- run one relationship rupture
- run one architectural repair
- measure whether the world visibly changes

## 15. Working thesis

Paramecia exceeds the assumptions of most traditional role-playing systems because it asks the game to simulate civilization rather than only adventure.

Its distinctive contribution is not merely a large setting. It is a world where relation, care, memory, architecture, and repair form one continuous engine.
