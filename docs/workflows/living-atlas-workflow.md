# Living Atlas Production Workflow

> Core truth: **Nothing flourishes alone.**

This workflow turns project-chat material into reviewed canon, versioned diagrams, and primary presentations without allowing exploratory material to silently harden into fact.

## 1. Capture

Collect exact phrases, corrections, images, region notes, character notes, systems, and unanswered questions from active Project 0 chats.

Each captured item begins as **Unreviewed**. Preserve the user's exact correction language before interpretation.

## 2. Shape in Notion

Notion is the readable working studio. Use it for:

- production queue and status
- region, character, institution, and system pages
- contradiction tracking
- research notes and presentation briefs
- diagram specifications
- output links

Shaping may clarify language, but must not overwrite unresolved contradictions.

## 3. Pass the Three Accord Gates

Every item must answer all three questions before canon lock:

### Recognition
**What is actually here?**

- What was explicitly stated?
- What is an inference?
- What conflicts with existing canon?
- What remains unknown?

### Harmony
**What wants to flourish?**

- What relationship, ecology, or possibility is this material supporting?
- Does it create room for plurality and agency?
- Does it strengthen the living system rather than merely decorate it?

### Midnight
**What must be honored?**

- What wound, limit, lineage, consent boundary, or cost must not be erased?
- What should remain difficult, unfinished, or protected?
- What cannot be simplified for presentation convenience?

## 4. Assign Canon Status

- **Unreviewed**: captured but not assessed
- **Exploration**: imaginative branch with no canon claim
- **Provisional**: coherent and usable, awaiting final confirmation
- **Canon**: explicitly confirmed and internally consistent
- **Contradiction**: conflicts with another active claim
- **Archive**: retained for history but no longer drives current work

## 5. Lock in GitHub

GitHub is the durable canon ledger.

Store:

- canonical prose in `canon/`
- workflow and method documents in `docs/`
- Mermaid and other source diagrams in `diagrams/`
- presentation outlines and references in `presentations/`
- unresolved changes as GitHub Issues

Major canon changes should use a branch and pull request when practical. Commit messages should name the actual change.

## 6. Route Diagram Work

Use the smallest tool that matches the relationship:

- **Mermaid in GitHub**: versioned flows, state diagrams, timelines, and entity relations
- **Beautiful.ai**: structured presentation diagrams such as processes, cycles, journeys, comparisons, Venn diagrams, funnels, and hub-and-spoke layouts
- **PowerPoint**: custom ecological atlas compositions and final presentation integration
- **Figma / FigJam**: selective visual experiments, editorial spreads, and occasional flow diagrams
- **Ace Knowledge Graph**: dense many-to-many lore networks after canon stabilization

Diagrams must preserve exact canonical labels. Decorative elegance does not outrank semantic accuracy.

## 7. Build the Primary Presentation

PowerPoint remains the master presentation artifact.

Production order:

1. select canon-locked material
2. define the narrative spine
3. identify which relationships require diagrams
4. generate or build reusable diagrams
5. add ecological artwork and atlas framing
6. review hierarchy, accessibility, and exact wording
7. export and release

Generated decks are drafts unless explicitly promoted.

## 8. Review and Publish

Before publication, verify:

- every slide has a unique purpose
- every diagram has an interpretable relationship
- speculative language is marked
- exact Accord wording is preserved
- Point 0 and other spatial claims match canon
- corrections from the user have not been softened away
- the artifact links back to its source files

Published outputs should be linked in Notion and, where useful, referenced under `presentations/`.

## 9. Recovery and Re-entry Rails

The workflow must protect the creator, not consume them.

- **Canon Surge**: capture exact phrases and corrections; separate canon from exploration
- **Drop Point**: pause, summarize what was captured, and name the next safe stop
- **Recovery Amber**: at least twenty minutes of no new work; receive mode only
- **Re-entry Corridor**: consolidate the last active project only; no new branches

A paused item is not a failed item. Care is infrastructure.
